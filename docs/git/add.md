# ***Команда git add***

## **Содержание**

### **Описание**

 Команда `git add` переносит изменения из рабочей директории в область подготовленных файлов (staging area). Она фиксирует текущее состояние файлов, готовя их к следующему коммиту. Это первый шаг сохранения изменений; они не попадают в репозиторий, пока не выполнена команда `git commit`. 

### **Распространенные опции**

- `git add <file>` - Добавляет конкретный файл (или файлы) в индекс.

- `git add .` - Добавляет все новые, измененные и удаленные файлы в текущей директории и поддиректориях.

- `git add -A (или --all)` - Добавляет вообще все изменения (новые, измененные, удаленные) во всем репозитории.

- `git add -p` - Интерактивное добавление частей файла, позволяющее выбрать, какие именно изменения (хуки) закоммитить.

- `git add -u` - Добавляет только измененные или удаленные файлы, но не новые. 
Команда используется для подготовки к коммиту и не работает с удаленными репозиториями.

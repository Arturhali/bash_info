# ***Команда git push***

### **Описание**

Команда `git push` используется в Git для отправки зафиксированных локальных изменений (коммитов) из локального репозитория в удаленный (например, на GitHub, GitLab). Она синхронизирует удаленную ветку с локальной, обновляя ее.

### **Распространенные опции**

- `-u или --set-upstream` -  Устанавливает связь между локальной веткой и удаленной. Используется при первой отправке новой ветки: `git push  -u origin <ветка>`.

- `-f или --force` - Принудительно перезаписывает историю на удаленном репозитории. Опасно, так как может удалить чужие коммиты.

- `--force-with-lease` - Более безопасный аналог `--force`. Принудительно отправляет изменения, только если удаленная ветка не была обновлена кем-то другим.

- `--all` - Отправляет все локальные ветки в указанный удаленный репозиторий.

- `--tags` - Отправляет теги (метки) в удаленный репозиторий.

- `--delete` - Удаляет указанную ветку или тег в удаленном репозитории: `git push origin --delete <ветка>`.

- `--prune` - Удаляет локальные ссылки на удаленные ветки, которые были удалены из источника.

- `-v или --verbose` - Выводит подробную информацию о процессе отправки.

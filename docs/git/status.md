# ***Команда git status***

## **Содержание**

1. [Описание](#описание)
2. [Распространенные опции](#распространеные-опции)

### **Описание**

Команда `git status` отображает состояние рабочего каталога и раздела проиндексированных файлов. С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git. Информация об истории коммитов проекта не отображается при выводе данных о состоянии.

### **Распространеные опции**

- `git status -s или --short` -  Выводит краткую информацию. Файлы обозначаются двумя буквами (статус индекса и рабочей директории), например, M — изменен, ?? — не отслеживается.

- `git status -b или --branch` -  Показывает название ветки и информацию об отслеживании (опережение/отставание от удаленной), обычно используется вместе с `-s` (например, `git status -sb`).

- `git status --ignored` - Включает в вывод файлы, которые попадают под правила .gitignore.

- `git status --show-stash` - Отображает количество записей в хранилище stash, если они есть.

- `git status --porcelain` - Выводит статус в скриптовом, неизменяемом формате, удобном для парсинга, независимо от настроек Git.

- `git status --untracked-files=all` - (по умолчанию) Показывает все неотслеживаемые файлы.

- `git status --untracked-files=no` - Скрывает неотслеживаемые файлы, показывая только изменения в отслеживаемых.
